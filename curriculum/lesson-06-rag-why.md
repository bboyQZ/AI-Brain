# Lesson 6：为什么需要 RAG？

> **第二阶段：RAG** — 聚焦 AI 工程师每天都在用的检索增强生成，不讲偏算法论文的内容。

## 本课目标

理解：

- 大模型为什么不知道你的业务知识？
- 为什么不用重新训练，而是先查资料再回答？
- RAG 到底是什么、整体流程长什么样？

---

## 第一节：为什么需要 RAG？

先问一个问题。

假设你问 ChatGPT：

> 原神是什么类型的游戏？

它基本都会回答：开放世界 ARPG。

**为什么？** 因为训练的时候学过。

但如果你问：

> AIHelp 最新 FAQ v4.3 的字段有哪些？

GPT 很可能不知道。

**为什么？** 因为**模型的大脑已经固定了**。

可以把 Transformer 想成一个人：

```
一个大学生 → 大学四年 → 毕业
```

毕业以后，大脑里的知识已经固定。以后：

| 新信息 | 他知道吗？ |
|--------|-----------|
| OpenAI 发布 GPT-6 | ❌ |
| AIHelp 改版 | ❌ |
| 你的公司数据库 | ❌ |

不是他笨，而是**没人告诉他**。

---

## 那怎么办？

很多人的第一反应：重新训练模型！

实际上几乎没人这么干。训练一次模型：

```
几百张 GPU + 几百万人民币 + 几周甚至几个月
```

只为了更新 FAQ、订单、SOP、员工手册——**太离谱了**。

---

## 一个生活例子

假设我问你：咱们公司测试流程是什么？

你不会重新读四年大学，而是：

```
打开公司 Wiki → 找到「测试规范.md」→ 看两分钟 → 回答
```

整个过程：

```
不知道 → 查资料 → 回答
```

LLM 其实也可以这样。

---

## GPT 以前 vs RAG 出现以后

**以前：**

```
用户：AIHelp 最新 FAQ 是什么？
  ↓
GPT：（我不知道）   ← 模型脑子里没有
```

**RAG 出现以后：**

```
用户提问
  ↓
去知识库找资料
  ↓
把资料交给 GPT
  ↓
GPT 回答
```

**RAG 本质就是给 LLM 加了一个「查资料」能力。**

---

## 举个 AIHelp 的例子

知识库文档：

```
FAQ v4.3
新增字段：分类、标签、优先级
```

用户问：FAQ 有哪些新字段？

| 方式 | 结果 |
|------|------|
| 纯 GPT | 不知道 |
| RAG | 检索到 FAQ 文档 → 阅读 → 回答 |

GPT 本身并没有变聪明。真正变的是：**它终于能看文档了。**

## RAG 到底是什么？

很多人第一次看到 **Retrieval-Augmented Generation**，感觉特别高级。

实际上可以翻译成一句非常普通的话：

> **一边查资料，一边回答。**

| 英文 | 含义 | 做什么 |
|------|------|--------|
| **Retrieval** | 检索 | 去找资料 |
| **Augmented** | 增强 | 给 GPT 增加信息 |
| **Generation** | 生成 | 生成回答 |

整个流程：

```
先检索 → 再增强 Prompt → 最后生成回答
```

---

## RAG 的整体流程（先有整体印象）

几乎所有 RAG 都可以画成下面这个图：

```text
                用户问题
                    │
                    ▼
          ① 把问题变成向量
                    │
                    ▼
         ② 去向量数据库检索
                    │
          找到最相关的几段文档
                    │
                    ▼
      ③ 拼到 Prompt（上下文）里
                    │
                    ▼
              发给大模型
                    │
                    ▼
                 生成答案
```

先不要着急理解每一步——后面几课会一层一层拆开。

---

## 今天最重要的认知

很多初学者误以为：**RAG 是一种新的模型。**

其实不是。RAG **不是模型**，也不是 Transformer 的升级版。它更像是：

```text
             GPT
              │
     ┌────────┴────────┐
     │                 │
 模型自己的知识      外部知识库
     │                 │
     └────────┬────────┘
              │
           综合回答
```

| 角色 | 职责 |
|------|------|
| **GPT** | 理解问题、推理、组织语言 |
| **RAG** | 把模型不知道的新知识送到 GPT 面前 |

> **RAG = 给大模型外挂了一个可以实时查阅的知识库。**

## 本课笔记

### 为什么需要 RAG？

- 大模型训练完成后，知识基本固定，无法自动知道公司最新文档、数据库或实时信息。
- 重新训练成本极高，不适合为少量业务知识更新而进行。
- 更高效的方法：**回答前先查资料，再根据资料回答。**

### RAG 是什么？

**RAG（Retrieval-Augmented Generation）** 的核心思想：

> **先检索（Retrieve），再生成（Generate）。**

它不是一种模型，而是一套**工程方案**。

### RAG 的基本流程

```text
用户问题 → 问题转向量 → 向量库检索 → 文档拼入 Prompt → LLM 阅读 → 生成答案
```

### 核心认识

- RAG 不会让 GPT 更聪明。
- 它只是让 GPT **拥有实时查阅外部知识的能力**。
- GPT 负责思考和表达，RAG 负责提供最新、最相关的知识。
