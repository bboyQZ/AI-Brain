# Lesson 3：什么是 Embedding 向量？

## 本课目标

理解：

- 向量到底是什么？
- 为什么 Embedding 要使用向量？
- 为什么语义相近的词，向量也会相近？

---

## 一句话理解

Embedding 本质上就是：

> **把每个 Token 放到一个高维数学空间中的一个位置（坐标）。**

模型并不是理解文字，而是在理解这些**坐标之间的关系**。

---

## 整体流程

```
文本
  ↓
Tokenizer
  ↓
Token ID
  ↓
Embedding
  ↓
高维向量（Vector）
  ↓
Transformer
```

Lesson 1 学了 Tokenizer，Lesson 2 学了 Embedding 的作用；本课深入理解 **向量本身** 是什么。

---

## 什么是向量？

Embedding 使用的向量，就是数学中的**向量**——一组有序的数字，表示空间中的一个点。

**二维坐标：**

```
(2, 3)
```

表示 x = 2，y = 3，这是一个**二维向量**。

**三维坐标：**

```
(2, 3, 5)
```

表示 x、y、z 三个维度，这是一个**三维向量**。

Embedding 也是一样，只是维度远高于 2 或 3：

| 常见维度 | 典型场景 |
|----------|----------|
| 768 | BERT、早期开源模型 |
| 1024 | 部分中文 Embedding 模型 |
| 1536 | OpenAI `text-embedding-3-small` |
| 3072 | OpenAI `text-embedding-3-large` |

**例如「北京」的 Embedding（示意）：**

```
(0.31, -0.72, 1.54, 0.08, ..., 共 768 个数字)
```

这就是「北京」这个 Token 在向量空间中的**坐标**。

---

## Token ID 为什么不能直接使用？

| 词 | Token ID（示意） |
|----|------------------|
| 北京 | 14002 |
| 上海 | 987 |

14002 和 987 只是编号，编号之间**没有任何语义**。

模型无法从数字大小推断：北京 ≈ 上海。

因此必须把 Token ID 转换成 Embedding 向量，才能在空间中比较「远近」。

---

## 为什么要使用高维向量？

现实世界中的一个词，包含很多隐含信息。

以「北京」为例，可能关联：

- 中国、城市、首都、北方、历史、旅游、政治……

**二维坐标无法表达这么多特征**，所以需要高维空间——维度越多，能编码的信息越丰富。

> **注意：** 这些维度**不是人为规定的**。
>
> - 第 1 维 ≠ 城市
> - 第 2 维 ≠ 国家
>
> 每一个维度的含义，都是模型在**训练过程中自动学习**出来的。目前没有人能准确解释「第 N 维代表什么」——我们只知道整体效果：**语义相近，向量相近**。

---

## 为什么语义相近的词，向量也会相近？

训练开始时，所有词的向量几乎都是**随机的**，彼此没有规律。

随着模型不断阅读海量文本：

- 「北京」「中国」「首都」「天津」「河北」经常一起出现
- → 「北京」与「上海」「广州」「深圳」的向量**距离越来越近**（城市聚类）

- 「香蕉」几乎不会与「北京市政府」「首都」一起出现
- → 「北京」与「香蕉」的向量**距离越来越远**

最终形成不同的**聚类（Cluster）**：

```
城市簇：北京 ≈ 上海 ≈ 广州 ≈ 深圳
水果簇：香蕉 ≈ 苹果 ≈ 西瓜
```

这就是「共现规律」驱动向量空间自我组织的过程。

---

## Embedding 最重要的一句话

> **语义相近，向量相近。**

模型真正利用的是**向量之间的距离**（或相似度），而不是 Token ID 的数值大小。

---

## 与 RAG 的关系

RAG **不会直接搜索文字**，而是搜索向量：

```
【离线】文档 → Embedding → 向量数据库

【在线】用户提问 → Embedding → 问题向量
                              ↓
              计算与知识库中所有向量的距离
                              ↓
                      找到最相近的内容
```

即使问题和文档**没有相同的关键词**，只要语义相近，RAG 依然能找到正确答案。

---

## 与实际工作的联系

**知识库原文：**

> 任务推送异常导致自动分配失败。

**用户提问：**

> 为什么客服没有自动接单？

两句话没有完全相同的关键词，但 Embedding 会认为：

```
「自动分配失败」 ≈ 「没有自动接单」
```

因此 RAG 可以正确检索到对应知识——这正是语义搜索相比关键词搜索的核心优势。

## 本课总结

| 组件 | 职责 |
|------|------|
| **Tokenizer** | 文本 → Token ID |
| **Embedding** | Token ID → 高维向量 |
| **Transformer** | 利用这些向量理解上下文，并进行推理 |

**Embedding 是连接「文字」与「模型理解」之间最重要的一层。**

## 我的理解（课后思考）

1. Token ID 只是编号，不包含语义。
2. Embedding 将编号映射为高维向量。
3. 向量的**位置**代表语义，而不是数字大小。
4. 模型通过比较向量之间的距离来判断语义是否相近。
5. RAG、向量数据库和语义搜索都建立在 Embedding 的基础之上。
