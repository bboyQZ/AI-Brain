# Chronicle 公司大楼 · 2.5D 错落翼楼设计

## 目标

将 Chronicle「公司大楼」视图从叠层玻璃长条，改为 **白底、2.5D 等轴、错落庭院翼楼**，内部可见真实比例工位与可定制 Q 版女生小人。

## 已确认决策

### 建筑

| 项 | 决策 |
|----|------|
| 布局 | **方案 A · 庭院翼楼**（非一条长平面） |
| 楼层 | 五部门同一层，左右错落；乔志老板间后中央抬高 |
| 翼楼 | 十秘（左前）、服务（中前）、文艺（右前）、管理（左内收）、清洁（后翼最大） |
| 连接 | 中庭 + 宽连廊（~2m） |
| 墙体 | 实体墙 + 地板；**朝相机侧外墙 + 屋顶透明**（动态剖面） |
| 比例 | 米制真实比例（工位间距 2.4–2.6m，部长区矮隔断，桌椅分离） |
| 视角 | 固定俯仰 ~35°；**左右拖拽绕 Y 轴旋转**（±75°）；滚轮缩放 |
| 背景 | 浅白 `#f4f6fa` |

### 角色 · 方案 A（轻量定制）

- 造型：**Q 版好看女生**（非写实），全员女生造型
- 共享 1 套基础身体几何 + `InstancedMesh`
- **发型**：8–12 种预设
- **服装**：6–10 种办公裙装/套装预设
- **配色**：发色、裙色、部门点缀色每人可不同（`flows.ts` 的 `appearance` 字段）
- 远景：简化 3D 小人；选中时侧栏可用现有头像/立绘
- 动画：仅 idle（呼吸/微动），无行走
- **性能目标**：50+ 人同屏桌面 60fps

### 交互

- 点击小人 → 选中 → 侧栏故事（与现有 Chronicle 一致）
- 拖拽平移、滚轮缩放、横向旋转

## Canvas 示意 vs 正式版

| | Canvas 示意 | Three.js 正式版 |
|--|-------------|-----------------|
| 建筑 | SVG 线框块 | 实体墙、玻璃 transmission、光影 |
| 人物 | 简笔三角裙 | Q 版 3D + 换装预设 |
| 旋转 | 滑块改剖面 | 真实 3D 轨道 |
| 目的 | 对齐平面布局 | 产品可交互视图 |

Canvas **无法**展示最终视觉，仅用于布局与比例沟通。

## 主要改动文件（实现时）

- `frontend/src/chronicle/orgLayout3d.ts` — 翼楼平面、工位坐标
- `frontend/src/chronicle/ChronicleOrgScene3D.tsx` — 场景重写
- `frontend/src/chronicle/orgCharacter.ts` — 小人工厂 + InstancedMesh
- `frontend/src/chronicle/orgFurniture.ts` — 桌椅、隔断、连廊
- `frontend/src/chronicle/flows.ts` — 可选 `appearance` 字段
- **不更新**源码导读（纯前端 Chronicle UI）

## 验收

1. 白底 2.5D，五翼错落非长条
2. 旋转时朝相机墙透明，内部工位可见
3. 清洁部左右分区、乔志后中央老板间
4. 女生小人按 appearance 显示不同发型/服装/配色
5. 点击选人、构建通过
