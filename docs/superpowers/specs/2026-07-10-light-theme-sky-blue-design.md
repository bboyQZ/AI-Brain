# AI-Brain 浅色主题（天蓝）设计文档

> 状态：已对齐，待实现规划  
> 日期：2026-07-10  
> 范围：整站视觉主题（对话 / 知识库 / 概念实验室）+ 浅/暗切换

---

## 1. 背景与目标

当前前端整站为暗色（黑底浅字），主色为紫色强调。用户希望整体改为**白底黑字**，气质接近 ChatGPT 的浅色产品感，并选定**天蓝**作为品牌强调色；同时保留可切换的暗色模式。

### 用户可见目标

1. 打开网站默认是浅色：白底黑字，侧栏淡天蓝，主内容区干净白底，强调色为天蓝  
2. 顶栏可切换到「深蓝夜空」暗色；再切回浅色  
3. 用户选择的浅/暗在刷新后仍然记住  
4. 对话、知识库、实验室三页都不再残留大块黑底白字  
5. 聊天、知识库、实验室的功能行为不变，只换视觉

### 非目标

- 不改聊天逻辑、后端 API、知识库内容  
- 不重做页面布局结构（仍为现有顶栏 + 各页现有分区）  
- 不跟随系统 `prefers-color-scheme` 自动变色（只听用户开关）  
- 不新增独立「外观设置」页  
- 不引入新的 UI 组件库或主题框架依赖

---

## 2. 已确认的视觉决策

| 决策点 | 结论 |
|--------|------|
| 默认主题 | 浅色 |
| 暗色 | 保留，可切换 |
| 浅色侧栏 | 柔和天空蓝底（方案 A：`#f0f9ff` 系） |
| 浅色主内容区 | 干净白底 + 天蓝强调（方案 B） |
| 暗色气质 | 深蓝夜空 + 同一套天蓝强调（与浅色品牌一致） |
| 覆盖范围 | 整站三页 + 顶栏；实验室 3D 背景随主题变 |

---

## 3. 架构方案（选定）

采用 **双套 CSS 变量 + `data-theme` 切换**：

1. 在 `frontend/src/index.css` 定义浅色（`:root` / `[data-theme="light"]`）与暗色（`[data-theme="dark"]`）两套语义变量  
2. 在 `html`（或根节点）上设置 `data-theme`  
3. 顶栏增加主题切换控件；选择写入 `localStorage`（键名建议：`ai-brain-theme`，值：`light` | `dark`）  
4. 首次访问无记录时默认 `light`  
5. 各组件 CSS 中写死的深色 `rgba` / 硬编码色，改为使用语义变量或基于 `--accent` 的透明色  
6. 实验室 Three.js 场景背景色读取当前主题对应的背景色（浅色用浅底，暗色用深蓝底）

未选方案：仅改浅色靠滤镜反转暗色；引入完整主题库。原因：无法精确落地「侧栏 A + 主区 B」与「深蓝夜空」，且对当前体量过重。

## 4. 设计 Token

### 4.1 浅色（默认）

| Token | 用途 | 值 |
|-------|------|-----|
| `--bg` | 主内容背景 | `#ffffff` |
| `--bg-elevated` | 顶栏、抬升面 | `#f8fafc` |
| `--sidebar-bg` | 会话侧栏等侧栏背景 | `#f0f9ff` |
| `--sidebar-border` | 侧栏分隔 | `#bae6fd` |
| `--bg-input` | 输入区表面 | `#ffffff` |
| `--border` | 通用边框 | `#e2e8f0` |
| `--text` | 主文字 | `#0f172a` |
| `--text-dim` | 次要文字 | `#64748b` |
| `--accent` | 主强调 / 主按钮 | `#0ea5e9` |
| `--accent-soft` | 选中/hover 浅底 | `rgba(14, 165, 233, 0.12)` |
| `--accent-glow` | 轻阴影/光感 | `rgba(14, 165, 233, 0.25)` |
| `--user-bubble` | 用户消息气泡 | `#e0f2fe` |
| `--assistant-bubble` | 助手消息底 | `transparent`（靠排版与间距区分，不另铺色块） |

品牌标题可用实色 `--accent` 或略深 `#0284c7`，不再使用紫色渐变。

### 4.2 暗色（深蓝夜空）

| Token | 用途 | 值 |
|-------|------|-----|
| `--bg` | 主背景 | `#0b1220` |
| `--bg-elevated` | 顶栏、抬升面 | `#0f1b2d` |
| `--sidebar-bg` | 侧栏 | `#0f1b2d` |
| `--sidebar-border` | 侧栏边框 | `#1e3a5f` |
| `--bg-input` | 输入区 | `#111827` |
| `--border` | 通用边框 | `#1e3a5f` |
| `--text` | 主文字 | `#e2e8f0` |
| `--text-dim` | 次要文字 | `#94a3b8` |
| `--accent` | 强调 | `#0ea5e9` |
| `--accent-soft` | 选中/hover | `rgba(14, 165, 233, 0.2)` |
| `--user-bubble` | 用户气泡 | `rgba(14, 165, 233, 0.22)` |
| `--assistant-bubble` | 助手区 | `transparent` |

暗色强调色与浅色同一色相，保证切换后仍是「天蓝品牌」，而不是回到旧紫黑。

### 4.3 保留的非色 Token

`--radius`、`--font`、`--mono` 保持现有值，本次不改排版体系。

## 5. 页面与组件覆盖

### 5.1 全局

- `index.css`：变量、`body` 背景/文字、顶栏、导航链接 hover/active  
- `App.tsx`：顶栏右侧主题切换；启动时读取 `localStorage` 并设置 `data-theme`

### 5.2 对话页

- `SessionSidebar.css`：使用 `--sidebar-bg` / `--sidebar-border`，选中态用 `--accent-soft`  
- `MessageList.css` / `MessageInput.css`：气泡、引用、发送按钮、输入卡片边框与阴影改为浅色友好变量  
- `ChatPage.css`：背景跟 `--bg`

### 5.3 知识库页

- `KnowledgePage.css`：列表、正文区、选中态、代码/引用块去掉写死深色半透明，统一走变量

### 5.4 概念实验室

- 各 `*Demo.css`、`LabPage.css`：面板背景用 `--bg-input` / `--bg-elevated`  
- `EmbeddingDemo.tsx`：Three.js `scene.background` 随主题更新（监听 `data-theme` 或传入当前背景色）

### 5.5 主题切换控件

- 位置：顶栏右侧（品牌与导航之后）  
- 交互：一键在浅色 / 暗色间切换  
- 文案：可用「浅色 / 暗色」或图标；需有可访问的 `aria-label`  
- 不新增设置页

---

## 6. 错误与边界

- `localStorage` 不可用时：仍可切换主题，但不持久化；默认浅色  
- 非法存储值：回退为 `light`  
- 切换主题时：无需刷新页面；CSS 变量即时生效；Three.js 背景需显式更新一次

---

## 7. 验收标准

1. 首次打开（无本地记录）为浅色：白底黑字，侧栏淡天蓝，主区白底，强调天蓝  
2. 顶栏可切到暗色：深蓝夜空，浅色文字，天蓝强调  
3. 再切回浅色正常；刷新后主题与上次选择一致  
4. `/`、`/lab`、`/knowledge` 三页均无大块旧紫黑底残留  
5. 发送消息、浏览知识、使用实验室演示功能行为与改前一致

---

## 8. 实现约束

- 优先改现有 CSS / 少量 `App.tsx` 与实验室背景逻辑，不新建平行主题系统文件（除非实现时发现单文件过大再拆）  
- 不引入新 npm 依赖做主题管理  
- 与既有 ChatGPT 风格布局改造（2026-07-08 对话页设计）兼容：本次只换色，不推翻已定布局
