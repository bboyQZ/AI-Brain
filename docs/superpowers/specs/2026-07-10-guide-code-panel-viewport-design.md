# 源码导读 UX 打磨设计（视口 + 右侧注释）

> 状态：已对齐，待实现规划  
> 日期：2026-07-10  
> 范围：`/guide` 流程图默认视口；右侧源码面板滚动与注释布局（A+C）

---

## 1. 背景与目标

当前 `/guide` 已具备：方法方块流程图（React Flow）+ 右侧真源码 + 行级讲解。近期反馈：

1. **默认进来应看整体**，疏朗布局优先；不必一屏把字放大到完全可读，用户可自行缩放。  
2. 右侧面板出现**左右滚动条**；右侧「大括号 + 旁注」列挤占宽度，底部「这段在干什么」与横滚体验别扭。  
3. 用户选定注释布局为 **A + C 组合**。

### 用户可见目标

1. 进入某条流程时，画布默认 **fitView 展示整图概览**。  
2. 方块间距保持疏朗美观；放大后字与说明可读。  
3. 右侧源码区**只有上下滚动**，长行自动换行，无左右滚动条。  
4. 每个讲解段：**白话注释在对应代码块上方**（A）。  
5. 面板顶部固定一条「这段在干什么」，随当前选中段切换（C）。  
6. 去掉现有右侧大括号旁注列，以及底部易与横滚搅在一起的摘要条（由顶部固定条替代）。

### 非目标

- 不改流程配置内容（`flows.ts` / `explains.ts` 文案可不动，除非布局需要微调）。  
- 不改后端 `/guide/source` 截取逻辑。  
- 不重做左侧流程切换与方块连线算法（正交折线与四向锚点可保留）。  
- 不引入新依赖。

---

## 2. 已确认决策

| 决策点 | 结论 |
|--------|------|
| 默认视口 | `fitView` 看整体 |
| 间距策略 | 疏朗优先；可读性靠缩放 |
| 注释布局 | A（段上注释）+ C（顶部固定当前说明） |
| 横向滚动 | 禁止；`pre-wrap` / `overflow-x: hidden` |
| 右侧大括号列 | 移除 |
| 底部 focus 条 | 移除，改由顶部固定条承担 |

---

## 3. 设计细节

### 3.1 流程图视口

- 切换流程或首次进入：对当前节点做 `fitView`（适度 `padding`，允许 `minZoom` 较低以便塞进整图）。  
- **不要**默认把视口 `setCenter` 到第一个节点且强制 `zoom: 1`。  
- 点击方块：仍可选中并加载右侧源码；**是否**顺带放大居中到该节点——默认**不强制放大**（避免打断「看整体」）；用户可用控件自行缩放。若实现时发现选中后找不到节点，可用小地图，不作为必须行为。  
- 左下角 Controls 保留「适应画布」；小地图保留。

### 3.2 右侧源码面板结构（自上而下）

1. 标题区：方法名、路径、函数级 `blurb`（现有 header / note 可保留）。  
2. **顶部固定条**（C）：标签「这段在干什么」+ 当前激活讲解段全文。无讲解时显示占位文案。  
3. **可上下滚动的代码区**：  
   - 按讲解分组渲染：  
     - 有讲解的组：上方注释卡片 → 下方该组代码行（可点击激活，同步顶部条）。  
     - 无讲解的行：普通代码行。  
   - 行号 + 代码两列即可；代码 `white-space: pre-wrap; word-break: break-word;`。  
   - 容器 `overflow-x: hidden; overflow-y: auto;`。

### 3.3 组件改动边界

| 文件 | 职责 |
|------|------|
| `frontend/src/guide/AnnotatedCode.tsx` | 改为 A+C 结构；去掉 brace 列与底部 focus |
| `frontend/src/pages/GuidePage.css` | 滚动与注释卡片样式；删除 brace 相关样式 |
| `frontend/src/pages/GuidePage.tsx` | 默认 `fitView`；去掉「进页强制 zoom=1 居中首节点」 |

---

## 4. 验收标准

1. 打开 `/guide` 任意流程：首屏能看到该流程**大部分或全部**方块（概览感），而非只盯着起点放大。  
2. 右侧面板拖宽/缩窄：代码区**不出现左右滚动条**；长行换行可见。  
3. 有多段讲解时：每段注释出现在对应代码**上方**；点击某段，**顶部固定条**文案变为该段说明。  
4. 暗色主题下注释卡片与顶部条仍可读。  
5. 点方块仍能加载真源码；无讲解的函数显示占位说明。

---

## 5. 风险与注意

- `fitView` 与疏朗间距并存时，概览字会偏小——这是已接受的取舍。  
- `pre-wrap` 后行号与视觉行可能不对齐「物理换行」——行号仍按源码逻辑行，可接受。  
- 顶部固定条 + 段上注释会有少量文案重复——有意为之（滚远时仍能看当前段）。
